3. S3 MEDIA STORAGE
================================================================================

Create S3 Bucket:
-----------------
1. AWS Console → S3 → Create bucket
2. Name: ai-content-agent-assets
3. Region: us-east-1
4. Uncheck "Block all public access"
5. Create bucket

Configure Bucket Policy:
------------------------
Bucket → Permissions → Bucket Policy:

{
    "Version": "2012-10-17",
    "Statement": [{
        "Effect": "Allow",
        "Principal": "*",
        "Action": "s3:GetObject",
        "Resource": "arn:aws:s3:::ai-content-agent-assets/*"
    }]
}

Configure CORS:
---------------
Bucket → Permissions → CORS:

[{
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": []
}]

Test S3 Upload:
---------------
python test_s3_upload.py

Upload Flow:
------------
User uploads → Validate → Upload to S3 → Store S3 URL in DynamoDB → Use for Instagram

File Validation:
----------------
Images: JPG, PNG (max 8MB)
Videos: MP4 (max 100MB)

4. EVENTBRIDGE + LAMBDA SCHEDULING
================================================================================

Architecture:
-------------
FastAPI → DynamoDB → EventBridge Rule → Lambda → Instagram → DynamoDB

Step 1: Create IAM Role
------------------------
1. IAM Console → Roles → Create role
2. Trusted entity: Lambda
3. Attach policies:
   - AWSLambdaBasicExecutionRole
   - Custom DynamoDB policy:

{
    "Version": "2012-10-17",
    "Statement": [{
        "Effect": "Allow",
        "Action": [
            "dynamodb:GetItem",
            "dynamodb:Query",
            "dynamodb:UpdateItem"
        ],
        "Resource": "arn:aws:dynamodb:*:*:table/ai_content_*"
    }]
}

4. Name: lambda-instagram-post-handler
5. Copy Role ARN

Step 2: Deploy Lambda
----------------------
export AWS_LAMBDA_ROLE_ARN=arn:aws:iam::ACCOUNT:role/lambda-instagram-post-handler
python deploy_lambda.py

Step 3: Configure Environment
------------------------------
Add to .env:
AWS_LAMBDA_POST_HANDLER_ARN=arn:aws:lambda:REGION:ACCOUNT:function:instagram-post-handler

Step 4: Test Lambda
-------------------
1. Lambda Console → instagram-post-handler
2. Create test event: {"post_id": "test-id"}
3. Click "Test"
4. Check CloudWatch logs

Scheduling Flow:
----------------
1. User schedules post via API
2. FastAPI creates DynamoDB record
3. FastAPI creates EventBridge rule
4. EventBridge triggers Lambda at scheduled time
5. Lambda posts to Instagram
6. Lambda updates DynamoDB status

Monitoring:

5. INSTAGRAM TOKEN REFRESH
================================================================================

Automatic Refresh:
------------------
✅ Happens automatically before posting
✅ Checks if token expires in < 7 days
✅ Refreshes token via Instagram API
✅ Updates token in DynamoDB
✅ No user action required

API Endpoints:
--------------
Check Status:
GET /auth/meta/instagram/token-status

Manual Refresh:
POST /auth/meta/instagram/refresh-token

Background Job Setup:
---------------------
Run daily to refresh all expiring tokens:

Windows Task Scheduler:
1. Open Task Scheduler
2. Create Basic Task
3. Name: "Instagram Token Refresh"
4. Trigger: Daily at 2:00 AM
5. Action: Start a program
6. Program: python
7. Arguments: token_refresh_job.py
8. Start in: C:\path\to\backend

Linux/Mac Cron:
crontab -e
0 2 * * * cd /path/to/backend && python token_refresh_job.py

Token Lifecycle:
----------------
1. User connects Instagram → Token issued (60 days)
2. Token stored in DynamoDB with expiry
3. Before posting → Check if expires in < 7 days
4. If yes → Refresh token (extends 60 days)
5. Background job → Refreshes all expiring tokens daily
6. If refresh fails → Mark as needs_reauth

Error States:
-------------
"needs_reauth" → User must reconnect Instagram
"expired" → Token expired, reconnect required
"invalid" → Token invalid, reconnect required

6. ANALYTICS SYSTEM
================================================================================

Architecture:
-------------
1. analytics_job.py - Fetches Instagram insights and stores in DynamoDB
2. agents/analytics_agent.py - Reads analytics and generates summaries
3. routes/analytics.py - API endpoints for accessing analytics

Setup Automatic Scheduling:
----------------------------
Run setup script:
> setup_analytics_cron.bat

This creates Windows Task Scheduler jobs at 8 AM and 8 PM daily.

Manual Execution:
-----------------
> python analytics_job.py

Or trigger via API:
POST /analytics/fetch

API Endpoints:
--------------
1. Get Campaign Analytics:
   GET /analytics/campaign/{campaign_id}
   
   Returns: impressions, reach, likes, comments, saves, shares, engagement_rate

2. Get Performance Summary:
   GET /analytics/campaign/{campaign_id}/summary
   
   Returns: Top posts, AI insights, recommendations

3. Get Post Analytics:
   GET /analytics/post/{post_id}
   
   Returns: Individual post metrics

How It Works:
-------------
1. Post published via Lambda → status="posted" + platform_post_id
2. analytics_job.py runs every 12 hours:
   - Finds all posts with status="posted"
   - Fetches Instagram insights
   - Stores in DynamoDB post_analytics table
3. Analytics Agent reads from DynamoDB:
   - Individual post analytics
   - Campaign aggregated analytics
   - Performance summaries with insights
4. Scheduler Agent uses historical analytics for optimal posting times

Important Notes:
----------------
- Instagram insights available 24 hours after posting
- Job skips posts without insights
- Analytics cached for 5 minutes
- Access tokens auto-refreshed if needed

Troubleshooting:
----------------
If analytics not appearing:
1. Check posts have status="posted" and platform_post_id
2. Verify Instagram access token valid
3. Wait 24 hours after posting
4. Check analytics_job.py logs
5. Manually run: python analytics_job.py

Common Issues:
--------------

1. "Access Denied" (S3):
   - Check bucket policy allows public read
   - Verify IAM permissions
   - Ensure ACL is public-read

2. "Lambda not triggered":
   - Check EventBridge rule exists
   - Verify Lambda ARN in rule target
   - Check Lambda permissions
   - Review EventBridge execution history

3. "Token refresh failed":
   - Check Instagram API accessible
   - Verify token not expired
   - Check network connectivity
   - Review Instagram API status

4. "Analytics not appearing":
   - Check posts have status="posted"
   - Verify Instagram access token valid
   - Wait 24 hours after posting
   - Check analytics_job.py logs

5. "DynamoDB table not found":
   - Run init_dynamodb.py
   - Check table names in .env
   - Verify AWS region correct
